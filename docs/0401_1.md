# [Uni Note] 백엔드 개발 현황 보고 (2026-04-01)

## 1. 개요
본 문서는 'Uni Note' 서비스의 핵심 기능인 로그인 및 대시보드 조회를 위한 백엔드 초기 구축 내용을 정리한 보고서입니다.

---

## 2. 데이터베이스 연동 및 환경 설정
- **데이터베이스**: MySQL (`uninote_db`)
- **접속 정보**: `root` 계정 및 포트 `3306` 사용
- **JPA 설정**: 
    - `ddl-auto: update`: 엔티티 변경 시 DB 테이블 자동 업데이트
    - `show-sql: true`: 실행되는 쿼리를 콘솔에 출력하여 디버깅 용이성 확보
- **JWT 설정**: 24시간 만료 시간 설정 및 비밀키 환경 설정 완료 (`application.yaml`)

---

## 3. 도메인 모델 (Entity) 설계 및 구현
데이터베이스 구조 설계(`db구조.md`)를 바탕으로 JPA 엔티티 클래스를 구현하였습니다.

### 3.1. 핵심 엔티티
- **Student (students)**: 학번(`studentNum`), 이름, 전공 정보를 관리합니다.
- **Professor (professors)**: 성함과 소속 학과 정보를 관리합니다.
- **Course (courses)**: 강의명, 학수번호를 관리하며 담당 교수와 1:N 관계를 가집니다.
- **Enrollment (enrollments)**: 학생과 강의 사이의 수강 정보를 관리하는 중간 테이블(N:M 관계 해소)입니다.

---

## 4. 보안 및 인증 시스템 (JWT)
보안성 확보와 세션 관리를 위해 JWT(JSON Web Token) 기반의 무상태(Stateless) 인증 체계를 구축하였습니다.

### 4.1. 주요 구성 요소
- **JwtUtil**: 토큰 생성, 검증 및 클레임(학번) 추출 로직 담당.
- **JwtFilter**: 모든 요청의 Header에서 JWT를 가로채 유효성을 검사하는 Spring Security Filter.
- **SecurityConfig**: 
    - 세션 미사용(Stateless) 설정.
    - CSRF 보안 비활성화.
    - `/api/auth/login` 경로에 대한 무조건적 허용 및 그 외 모든 요청에 대한 인증 강제.

---

## 5. 로그인 API 구현
사용자 인증 및 토큰 발급을 위한 최초 API를 개발하였습니다.

- **엔드포인트**: `POST /api/auth/login`
- **입력 데이터**: `LoginRequest` (학번, 비밀번호)
- **응답 데이터**: `LoginResponse` (JWT 토큰, 학번)
- **처리 로직**: 
    1. `StudentRepository`를 통해 입력받은 학번의 존재 여부 확인.
    2. 학생이 존재할 경우 `JwtUtil`을 사용하여 해당 학번 기반의 토큰 생성 및 반환.
    *(현재 단계에서는 초기 개발 편의를 위해 비밀번호 검증 없이 학번 일치 여부만 확인합니다.)*
