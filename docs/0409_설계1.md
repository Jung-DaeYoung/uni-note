# [설계안] 스마트 강의 상세 페이지 (Course Detail Page)

**작성일**: 2026-04-09
**상태**: 초안 (Draft)

## 1. 개요
강의 대시보드에서 특정 강의를 선택했을 때 진입하는 페이지로, 필기 기능과 소통 기능을 한 화면에서 유기적으로 제공하는 것을 목표로 함. AI 기술을 접목하여 단순 기록을 넘어선 학습 보조 도구로서의 역할을 수행함.

## 2. UI/UX 레이아웃 설계

### 2.1. 3단 가변 레이아웃 (Tri-pane Layout)
1. **Left (Global Nav)**: LMS 메뉴 (대시보드 이동, 프로필 등). 좁은 너비(80px~260px)로 토글 가능.
2. **Center (Main Editor)**: 필기 노트 영역. 화면의 약 70~100%를 차지 (게시판 상태에 따라 가변).
3. **Right (Sidebar Board)**: 익명 게시판 영역. 약 30% 너비를 차지하며, 우측으로 슬라이드하며 사라지거나 나타남.

### 2.2. 주요 컴포넌트 배치
- **상단 툴바 (Header)**:
    - 현재 강의명 및 교수명 표시.
    - AI 도구 버튼: `✨ 요약하기`, `📝 문제생성`.
    - 게시판 토글 버튼: `[게시판 열기/닫기]`.
- **필기 영역 (Editor)**:
    - 구조화된 템플릿(학습 목표, 주요 내용, 질문) 기반 에디터.
    - 하단에 'AI 요약 결과' 및 '자동 저장 상태' 표시바 배치.
- **게시판 영역 (Board)**:
    - 실시간 익명 피드 리스트.
    - 하단 고정형 질문 입력창.

## 3. 기능 상세 설계

### 3.1. 스마트 필기 템플릿
- **템플릿 구조**:
    - **[학습 목표]**: 강의 전 목표 설정.
    - **[주요 내용]**: 메인 필기 섹션 (Rich Text 지원).
    - **[질문 리스트]**: 학습 중 발생한 의문점 기록.
- **자동 저장**: 2초간 입력이 없을 시 현재 내용을 백엔드 API(`/api/notes/save`)로 전송.

### 3.2. AI 학습 에이전트
- **요약 기능 (Summarize)**:
    - 에디터의 텍스트를 추출하여 Gemini API에 전송.
    - 프롬프트: "다음 강의 필기 내용을 바탕으로 핵심 키워드 5개와 3문장 요약을 작성해줘."
    - 결과를 별도의 하단 패널에 출력.
- **문제 생성 (Quiz Generation)**:
    - 필기 내용을 기반으로 예상 문제 3문항 생성.
    - 형식: JSON (질문, 보기, 정답, 해설).
    - 모달(Modal) 창을 통해 퀴즈 진행 UI 제공.

### 3.3. 열고 닫는 익명 게시판
- **동작 방식**:
    - `isBoardOpen` 상태값(Boolean)에 따라 CSS 클래스(`open`/`closed`) 부여.
    - `transition: transform 0.3s ease-in-out`을 사용하여 부드러운 애니메이션 구현.
- **익명성**:
    - 작성자 이름을 무작위 아이콘과 '익명 + 숫자' 조합으로 표시.

## 4. 데이터 인터페이스 (API 예상)

| 기능 | 엔드포인트 | 메소드 | 요청 파라미터 |
| :--- | :--- | :--- | :--- |
| 노트 저장 | `/api/notes` | POST | `{ courseId, content }` |
| 노트 조회 | `/api/notes/{courseId}` | GET | - |
| AI 요약 | `/api/ai/summarize` | POST | `{ content }` |
| AI 문제생성 | `/api/ai/quiz` | POST | `{ content }` |
| 게시글 목록 | `/api/posts/{courseId}` | GET | - |
| 게시글 작성 | `/api/posts` | POST | `{ courseId, title, content, isAnonymous: true }` |

## 5. 향후 확장성
- 필기 내용 중 모르는 용어 드래그 시 즉시 검색/설명 기능 추가.
- 다크 모드(Dark Mode) 지원을 통한 시력 보호.
- 실시간 동시 접속자 수 표시 (게시판 상단).
