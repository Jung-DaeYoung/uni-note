# 2026-04-11 작업 요약 (0411_1.md)

## 1. 익명 게시판(Post) 기능 고도화
오늘 작업의 핵심은 강의별 익명 게시판에서 사용자가 게시글을 자유롭게 작성, 삭제하고, 항상 최신 글을 먼저 볼 수 있도록 하는 기능을 구현한 것입니다.

### 1) 백엔드 (Spring Boot) 상세 구현
- **도메인 모델 (`Post`)**:
    - `title`, `content`, `author`(익명 닉네임) 필드를 포함하며, `Course` 엔티티와 `@ManyToOne` 관계를 맺어 강의별 게시판 구조를 확립했습니다.
    - `BaseTimeEntity`를 상속받아 생성 및 수정 시간을 자동 관리합니다.
- **최신순 정렬 로직 (`PostRepository`)**:
    - `findAllByCourseIdOrderByCreatedAtDesc` 쿼리 메서드를 구현하여, DB 조회 단계에서부터 생성일 기준 내림차순으로 정렬된 데이터를 가져오도록 최적화했습니다.
- **API 컨트롤러 (`PostController`)**:
    - `POST /api/posts`: 새 게시글 등록. `SecurityContext`를 통해 현재 사용자의 권한을 확인합니다.
    - `DELETE /api/posts/{postId}`: 게시글 ID를 기반으로 삭제 요청을 처리하며, 본인 확인 로직을 포함합니다.
    - `GET /api/posts/course/{courseId}`: 특정 강의의 게시글 목록을 최신순으로 반환합니다.

### 2) 프런트엔드 (React + Tailwind CSS) 연동
- **실시간 목록 업데이트**:
    - 새 게시글 작성 후 서버 응답이 성공하면, 전체 목록을 다시 불러오지 않고 기존 `posts` 상태값의 맨 앞에 새 게시글을 추가(`[newPost, ...posts]`)하여 즉각적인 피드백을 제공합니다.
- **삭제 인터랙션**:
    - 삭제 버튼 클릭 시 API 호출 후, `filter` 함수를 사용하여 UI에서 해당 항목을 즉시 제거하는 낙관적 UI 업데이트 방식을 적용했습니다.
- **시간 표시 및 UI**:
    - 최신순으로 정렬된 리스트를 렌더링하며, 각 게시글의 작성 시간을 사용자 친화적으로 표시할 수 있도록 구성했습니다.

## 2. 핵심 성과
- **최신 게시글 우선 노출**: 정렬 로직을 서버와 클라이언트 양쪽에서 견고하게 처리하여 게시판의 활성도를 높였습니다.
- **매끄러운 UX**: 페이지 새로고침 없는 CRUD 처리를 통해 네이티브 앱과 같은 사용성을 확보했습니다.
