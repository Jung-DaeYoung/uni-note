# [UniNote] 2026-04-13 개발 일지

## 1. 주요 수정 사항 및 해결된 문제

### 1.1. 로그인 기능 장애 해결
*   **원인 분석**:
    *   `AuthService`에서 비밀번호를 평문으로 비교하고 있었으나, `GlobalExceptionHandler`에서 발생한 예외를 `404 Not Found`로 반환하여 프런트엔드에서 원인 파악이 어려웠음.
    *   데이터베이스 `students` 테이블의 PK 컬럼명(`stud_id`)과 JPA 엔티티의 설정(`student_id`)이 일치하지 않아 `SQLSyntaxErrorException` 발생.
*   **해결 내용**:
    *   `GlobalExceptionHandler`: 로그인 실패 시 예외 처리 코드를 `401 Unauthorized`로 변경하여 인증 실패를 명확히 함.
    *   `AuthService`: 비밀번호 비교 시 `trim()`을 추가하여 공백 문제를 방지하고, 상세 디버깅 로그를 추가함.
    *   `Student` 엔티티: PK 컬럼명을 `stud_id`로, 학번 컬럼명을 `student_num`으로 실제 DB에 맞춰 수정.

### 1.2. 전수 데이터베이스 스키마 정합성 맞춤
*   **문제**: 엔티티 클래스들과 `docs/db구조.md` 문서, 실제 MySQL 테이블 간의 컬럼명 불일치 발견.
*   **수정 사항**:
    *   `stud_id`를 모든 테이블(Student, Enrollment, Post, Note, Comment)의 기준 FK 명칭으로 통일.
    *   `Professor`: `professor_id` → `prof_id`
    *   `Course`: `courseName` → `course_name`, `courseCode` → `course_code` 명시적 매핑.
    *   `Enrollment`: `enrollment_id` → `enroll_id`

### 1.3. DTO 및 서비스 로직 고도화
*   **익명 게시판 본인 확인**: `PostResponse`와 `CommentResponse`에 `isAuthor` 필드 추가.
*   **서비스 업데이트**: `PostService`와 `DashboardService`에서 현재 로그인한 사용자와 작성자를 비교하여 `isAuthor` 값을 채워주도록 로직 수정.
*   **불필요 필드 제거**: 설계 변경으로 사용하지 않게 된 필드들을 제거하여 코드 경량화.
    *   `Post`: `isAnonymous` 제거 (무조건 익명 고정 정책).
    *   `Note`: AI 요약(`aiSummary`), AI 퀴즈(`aiQuizData`) 필드 제거.
    *   `Student`: 전공(`major`) 필드 제거.

### 1.4. 프런트엔드 UI/UX 개선
*   **게시글 삭제 권한**: `isAuthor` 필드를 활용하여 본인이 쓴 글에만 삭제 버튼(`Trash2`)이 보이도록 수정.
*   **사이드바 최적화**: 왼쪽 사이드바 너비를 `256px`에서 `200px`로 줄여 메인 콘텐츠 영역 확보.
*   **노트 영역 확장**: 강의 상세 페이지에서 'Main Content' 헤더와 아이콘을 제거하고 텍스트 영역을 최대로 확장.
*   **헤더 정리**: 현재 사용하지 않는 상단 검색창 제거.
