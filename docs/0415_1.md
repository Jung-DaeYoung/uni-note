- 익명 게시판 및 댓글의 **수정/삭제 기능** 설계 및 구현 완료.
- 백엔드와 프런트엔드 간의 **데이터 정합성(isAuthor) 문제** 최종 해결.
- 데이터베이스 엔티티 및 테이블 구조 최적화.

## 🛠 상세 작업 내용

### 1. 게시글 관리 기능 고도화 (수정/삭제)
- **백엔드**: 
    - `PostService.java`에 `updatePost`, `deletePost` 메서드 추가.
    - 작성자 본인 확인 로직 강화 (`.trim()`을 사용한 학번 비교).
- **프런트엔드**:
    - 게시글 상세 보기에서 본인 글일 경우 수정(연필), 삭제(휴지통) 아이콘 노출.
    - 수정 모드(`VIEW: EDIT`) UI 구현 및 서버 연동.

### 2. 댓글 관리 기능 구현 (수정/삭제)
- **백엔드**:
    - `PostController.java`에 댓글 전용 수정/삭제 API 엔드포인트 추가.
    - `PostService.java`에 `updateComment`, `deleteComment` 구현.
- **프런트엔드**:
    - 각 댓글 우측에 인라인 수정/삭제 버튼 추가.
    - 댓글 수정 시 페이지 이동 없이 그 자리에서 편집 가능한 **인라인 에디터** 구현.
    - 수정/삭제 성공 시 실시간 데이터 동기화 로직 보강.

### 3. 데이터 통신 및 직렬화 문제 해결 (Critical)
- **이슈**: 백엔드에서 `isAuthor: true`를 보내도 프런트엔드에서 버튼이 보이지 않음.
- **원인**: 
    1. Java의 `boolean isAuthor` 필드가 JSON 변환 시 Jackson 라이브러리에 의해 `author`로 자동 변경됨.
    2. 학번 비교 시 미세한 공백 차이로 인해 본인 여부 판정 오류 발생.
- **해결**:
    - 백엔드 DTO에 `@JsonProperty("isAuthor")` 명시하여 필드명 고정.
    - 문자열 비교 시 `.trim()` 적용.
    - 프런트엔드에서 `isAuthor`와 `author` 필드를 동시에 체크하는 방어 로직 적용.

### 4. DB 및 엔티티 최적화
- `notes` 테이블에서 불필요한 `is_anonymous` 속성 제거 결정 (강의 노터는 개인 자산이므로 익명성이 불필요함).
- `posts` 테이블의 익명 관리 필드를 `anonymous` (TINYINT 1/0)로 단일화.
