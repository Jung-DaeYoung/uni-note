# UniNote 개발 일지 (2026-04-28)

오늘 작업은 기존의 단순한 입력 방식을 탈피하여, **노션(Notion) 스타일의 지능형 에디터 구축**과 **안정적인 데이터 저장 시스템** 확보에 집중했습니다.

## 1. 지능형 에디터(Tiptap) 도입 및 UI 시각화
- **기술 스택**: Tiptap 엔진을 기반으로 UniNote 전용 에디터 구축.
- **디지털 종이 질감**: Tailwind CSS v4를 활용하여 부드러운 그림자와 종이 질감이 느껴지는 캔버스 레이아웃 적용.
- **가독성 최적화**: 대학 강의 노트에 최적화된 서체, 행간, 여백을 적용하여 가독성 향상.

## 2. 2단계 자동 저장 및 데이터 복구 시스템
- **LocalStorage (즉각)**: 브라우저 임시 저장소를 활용해 입력 즉시 데이터를 기록, 예기치 못한 종료 시 유실 방지.
- **Debounce API (지연)**: 입력 중단 2초 후 백엔드 서버로 구조화된 JSON 데이터 전송 (영구 보존).
- **지능형 복구**: 페이지 로드 시 서버와 로컬 데이터를 비교하여 가장 최신 상태를 자동으로 복원하는 로직 구현.

## 3. 노션 스타일 상호작용 (UX)
- **버블 메뉴 (Bubble Menu)**: 텍스트 드래그 시 서식 도구(Bold, Italic, Code 등) 및 'Ask AI' 버튼 노출.
- **슬래시 명령어 (Slash Command)**: `/` 입력 시 제목, 체크리스트, 코드 블록 등을 선택할 수 있는 플로팅 메뉴 구현.
- **키보드 제어**: 슬래시 메뉴에서 화살표 키로 이동하고 엔터로 선택하는 키보드 네비게이션 기능 보완.

## 4. 코드 리팩토링 및 구조 최적화
- **컴포넌트 분리**: 비대해진 `CourseDetailPage.jsx`에서 에디터 로직을 `TiptapEditor.jsx`로 완전히 분리하여 유지보수성 향상.
- **상태 관리**: 에디터 내부에 저장 상태 배지(Synced, Saving, Error)를 통합하여 직관적인 피드백 제공.

## 5. 향후 과제
- **AI 에이전트 연동**: 필기 내용을 바탕으로 실시간 요약 및 핵심 키워드 추출.
- **CBT(퀴즈) 시스템**: 노트 데이터를 기반으로 한 문제 생성 및 학습 모드 구현.
