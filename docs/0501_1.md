# 2026-05-01 개발 작업 기록 (이미지 업로드 시스템 및 에디터 개선)

## 1. 개요
Tiptap 에디터에 독립 블록 형태의 이미지 업로드 기능을 추가하고, 이를 서버(Spring Boot)와 연동하여 안정적으로 저장 및 표시하는 시스템을 구축함.

## 2. 주요 작업 내용

### 2.1 백엔드 (Spring Boot & MySQL)
- **이미지 업로드 API 구현**:
    - `POST /api/upload/image`: MultipartFile을 받아 서버 로컬 저장소(`uploads/`)에 UUID 기반 파일명으로 저장.
    - 저장 후 브라우저에서 접근 가능한 이미지 URL(`/uploads/filename.png`) 반환.
- **정적 리소스 핸들러 설정**:
    - `WebConfig`: 외부 호출 경로 `/uploads/**`와 실제 서버 파일 경로를 매핑하여 이미지 서빙 활성화.
- **보안 설정(SecurityConfig) 최적화**:
    - **CORS 설정**: 프론트엔드(Port 5173)와의 통신을 위해 전용 `CorsConfigurationSource` 적용.
    - **인가 정책**: 업로드 API 및 리소스 경로에 대해 `permitAll()` 설정.
    - **오류 분석**: 403 Forbidden 해결을 위해 `/error` 경로 개방 및 Axios 수동 헤더 제거 가이드 적용.
- **데이터베이스 엔티티 확장**:
    - `Note` 엔티티에 `title`, `previewText`, `searchContent` 필드 추가하여 향후 검색 및 대시보드 기능 확장 대비.
- **설정 최적화**:
    - `application.yaml`: 파일 업로드 용량 제한을 10MB로 상향.

### 2.2 프론트엔드 (React & Tiptap)
- **Image Block 기능 추가**:
    - Tiptap Image Extension 적용 및 블록 단위 관리 구조 구축.
- **실제 서버 업로드 연동**:
    - Base64(FileReader) 방식 제거 -> `axios` + `FormData` 기반 서버 업로드 로직으로 교체.
    - 드래그 앤 드롭(Drag & Drop) 시 자동 업로드 및 서버 URL 기반 이미지 삽입 구현.
- **Slash Command 개선**:
    - `"/이미지 업로드"` 명령어 추가 및 로컬 파일 탐색기 연동.
    - 메뉴 목록이 길어질 때 키보드 방향키 이동에 따른 **자동 스크롤(scrollIntoView)** 기능 추가.
- **UI/UX 최적화**:
    - 에디터 내 이미지 최대 너비를 `650px`로 제한하여 본문 가독성 확보.
    - 업로드 중 상태 표시(Saving/Synced) 연동.

## 3. 해결된 주요 이슈
- **403 Forbidden**: 
    - 원인 1: Spring Security의 인가 정책 미설정.
    - 원인 2: Axios에서 `Content-Type` 수동 지정 시 `boundary` 누락 문제 -> 헤더 제거로 해결.
    - 원인 3: CORS Preflight(OPTIONS) 차단 -> Security 필터 체인 최적화로 해결.
- **이미지 깨짐**: 서버 주소(`http://localhost:8080`)를 누락한 상대 경로 문제를 절대 경로 삽입 방식으로 해결.
- **YAML 중복 키**: `spring:` 키가 두 번 선언되어 서버 부팅이 안 되던 문제 해결.

## 4. 향후 계획
- 저장된 이미지 블록을 활용한 OCR(텍스트 추출) 기능 검토.
- 이미지 크기 조절(Resizing) 핸들 추가 검토.
- 업로드된 미사용 이미지 파일 정리(Cleanup) 로직 구현 예정.
