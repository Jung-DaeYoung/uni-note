# 2026년 5월 2일 개발 일지

## 1. 에디터 오류 수정 및 안정화

### 1.1. JSX 파싱 및 확장자 문제 해결
- **문제**: `PageLink.js` 파일에 JSX 구문이 포함되어 Vite 빌드 중 파싱 에러 발생. 또한 동일한 내용의 `.js`와 `.jsx` 파일이 중복 존재하여 혼선 발생.
- **해결**: 중복된 `PageLink.js` 파일을 삭제하고, `NotionEditor.jsx`에서의 임포트 경로를 `.jsx`로 명시하여 정확한 파일을 참조하도록 수정.

### 1.2. Tiptap NodeView 규칙 준수
- **문제**: `PageLink` 커스텀 노드가 Tiptap의 React 렌더링 규칙인 `NodeViewWrapper`를 사용하지 않아 런타임 에러 발생.
- **해결**: `PageLink.jsx`의 최상위 요소를 `<NodeViewWrapper>`로 감싸 Tiptap이 DOM을 정상적으로 제어할 수 있도록 수정.

### 1.3. React 18 라이프사이클 경고(`flushSync`) 해결
- **문제**: `useEffect` 내에서 에디터 콘텐츠를 설정(`setContent`)할 때 React의 렌더링 주기와 Tiptap의 동기적 업데이트가 충돌하여 경고 발생.
- **해결**: `useEditor`의 `content` 옵션을 사용하여 에디터 초기화 시점에 데이터를 주입하는 방식으로 구조 개선. `setTimeout`을 통한 회피가 아닌 정석적인 초기화 방식으로 변경.

### 1.4. 빈 텍스트 노드 에러(`RangeError`) 해결
- **문제**: 제목이 없는 노트를 로드할 때 `{ type: 'text', text: "" }`와 같은 빈 텍스트 노드가 생성되어 ProseMirror 스키마 위반 에러 발생.
- **해결**: 초기 콘텐츠 생성 로직에서 텍스트가 있을 때만 `text` 노드를 생성하고, 없을 경우 부모 노드의 `content`를 빈 배열로 설정하도록 조건부 로직 적용.

### 1.5. Tippy.js 클린업 로직 보강
- **문제**: React StrictMode나 빠른 입력으로 인해 `onExit`가 중복 호출되면서 이미 파괴된 Tippy 인스턴스에 다시 `destroy()`를 호출하는 경고 발생.
- **해결**: `onExit` 내에서 인스턴스 파괴 후 참조를 즉시 `null`로 할당하는 방어적 코드 적용.

## 2. 네비게이션 및 데이터 동기화 개선

### 2.1. 노트 전환 시 잔상 문제 해결
- **문제**: 사이드바에서 다른 노트를 클릭할 때, 새 데이터를 불러오는 동안 이전 노드의 내용이 잠깐 표시되는 현상 발생.
- **해결**: 
  - `CourseDetailPage`에서 `noteId` 변경 시 `noteData`를 즉시 초기화.
  - 데이터의 ID와 현재 URL의 ID가 일치할 때만 에디터를 렌더링하도록 조건부 렌더링 강화.
  - `NotionEditor` 내부에서 데이터 로딩 완료를 감지하여 즉시 반영하도록 `useEffect` 의존성 보강.

### 2.2. 노트 링크 제목 실시간 동기화 구현
- **문제**: 에디터 내에 삽입된 `PageLink` 블록의 제목이 연결된 실제 노트의 제목 변경을 반영하지 못하고 과거 시점에 머물러 있는 문제.
- **해결**:
  - `NoteTreeContext`를 신설하여 애플리케이션 전체에 최신 노트 트리 데이터 공급.
  - `PageLink` 컴포넌트가 저장된 정적 속성 대신 컨텍스트에서 `noteId`를 기반으로 최신 제목을 실시간으로 조회(`Dynamic Lookup`)하도록 수정.
  - 이를 통해 사이드바나 다른 곳에서 제목을 수정하면 에디터 내의 모든 관련 링크 제목이 즉시 업데이트됨.
