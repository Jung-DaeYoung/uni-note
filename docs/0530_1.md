# 작업 일지 (2026-05-30)

## 1. AI 퀴즈 출처 추적 (Origin Tracking) 기능 구현
- **백엔드 고도화**:
  - `QuizService.java`에서 텍스트 추출 시 `[[REF:noteId/blockId]]` 메타데이터 주입 로직 추가.
  - Gemini AI 프롬프트를 수정하여 각 문항의 정답 근거가 된 `sourceBlockId`를 정확히 추출하도록 개선.
- **프런트엔드 네비게이션**:
  - `CBTPlayer.jsx` 결과 화면에 '원문 보기' 버튼 추가.
  - `NotionEditor.jsx`에 특정 블록 ID 기반 자동 스크롤 및 하이라이팅 애니메이션(`origin-highlight`) 구현.

## 2. 학습 결과 리포트 재조회 시스템 구축
- **데이터 영속화**:
  - `QuizAttempt` 및 `UserAnswer` 엔티티를 활용하여 퀴즈 풀이 결과를 DB에 영구 저장.
  - `POST /api/quiz/attempts` 등 관련 REST API 개발.
- **CBTPlayer 기능 확장**:
  - `'solve'`(풀이) 모드와 `'report'`(조회) 모드를 지원하도록 컴포넌트 구조 개편.
  - 과거 답안 데이터를 주입받아 풀이 당시의 화면을 재현하는 로직 구현.
- **학습 보관함(QuizLibraryPage) 고도화**:
  - '문제 모음'과 '풀이 기록' 탭 시스템 도입.
  - 각 퀴즈 카드에 **[결과 확인]** 버튼을 추가하여 해당 퀴즈의 역대 성적을 모달로 즉시 확인 가능하도록 통합.

## 3. 버그 수정 및 안정성 강화
- **ID 유실 문제 해결**: 퀴즈 생성 직후 문항에 DB ID(`questionId`)가 누락되어 저장이 실패하던 버그 수정.
- **NullPointerException 방어**: 
  - 과거 기록 정렬 시 `startTime`이 null인 경우 대비 로직 추가.
  - 원본 문항 삭제 시에도 리포트 조회가 가능하도록 예외 처리.
- **Import 경로 수정**: `QuizAttemptsModal.jsx`의 `client.js` 경로 오류(`../../` -> `../../../`) 수정.
