# [Uni Note] 디렉토리 구조 (Directory Structure)

본 문서는 Uni Note 프로젝트의 전체 디렉토리 구성 및 각 폴더의 역할을 정의합니다.

---

## 1. 전체 구조 (Overview)

```text
capstone/
├── docs/                 # 프로젝트 문서 (명세서, 설계서 등)
├── frontend/             # React 기반 프론트엔드 소스
└── backend/              # Spring Boot 기반 백엔드 소스
```

---

## 2. 상세 구조 (Detailed Structure)

### 2.1. 프로젝트 문서 (docs/)
기존의 명세서 및 설계서 파일을 관리하는 폴더입니다.
- `요구사항 명세서.md`
- `세부기능 명세서.md`
- `rest api 설계.md`
- `디렉토리구조.md`

### 2.2. 프론트엔드 (frontend/)
React.js 환경의 폴더 구조입니다.
- `public/`: 정적 리소스 (index.html, favicon 등)
- `src/`: 핵심 소스 코드
    - `api/`: Axios 인스턴스 및 백엔드 API 호출 함수 (services)
    - `assets/`: 이미지, 아이콘, 폰트 등 정적 파일
    - `components/`: 재사용 가능한 UI 컴포넌트
        - `common/`: Header, Sidebar, Button, Layout 등
        - `dashboard/`: 강의 카드, 히트맵 컴포넌트 등
        - `note/`: 에디터, 히스토리 리스트 등
        - `board/`: 게시글 목록, 댓글 리스트 등
    - `hooks/`: 커스텀 Hook (인증, 데이터 페칭 등)
    - `pages/`: 라우팅되는 주요 페이지 (Login, Dashboard, NoteDetail, BoardDetail)
    - `store/`: 전역 상태 관리 (Context API 또는 Redux Toolkit)
    - `styles/`: 전역 CSS 및 스타일 관련 파일
    - `utils/`: 공통 유틸리티 함수 (날짜 포맷터 등)
    - `App.js`: 메인 라우팅 설정
    - `index.js`: 엔트리 포인트

### 2.3. 백엔드 (backend/)
Spring Boot 환경의 표준 폴더 구조입니다.
- `src/main/java/com/uninote/`
    - `config/`: 보안(Spring Security), JWT, CORS 등 설정 파일
    - `controller/`: REST API 엔드포인트 구현
    - `dto/`: 데이터 전송 객체 (Request/Response DTO)
    - `entity/`: JPA 엔티티 클래스 (Database Table Mapping)
    - `repository/`: Spring Data JPA 레포지토리 인터페이스
    - `service/`: 비즈니스 로직 및 트랜잭션 처리
    - `security/`: JWT 필터, 인증 공급자 등 보안 관련 로직
    - `exception/`: 공통 에러 핸들링 및 커스텀 예외 클래스
- `src/main/resources/`
    - `application.yml`: 서버 및 DB 접속 설정
    - `static/` & `templates/`: 정적 리소스 (필요 시)
- `build.gradle`: 빌드 및 의존성 설정 파일

---

## 3. 관리 가이드
- **파일명 컨벤션**: 
    - Frontend: React 컴포넌트는 `PascalCase`, 일반 함수는 `camelCase`를 권장합니다.
    - Backend: Java 클래스는 `PascalCase`, 패키지명은 `lowercase`를 따릅니다.
- **환경 변수**: 보안이 필요한 정보(API Key, JWT Secret 등)는 `.env` (Frontend) 및 `application-secrets.yml` (Backend)에 관리하며, Git에는 업로드하지 않습니다.
