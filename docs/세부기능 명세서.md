# [Uni Note] 세부기능 명세서 (Detailed Functional Specification)

## 1. 개요
본 문서는 'Uni Note' 프로젝트의 요구사항 명세서를 바탕으로, 각 기능의 상세 동작 로직, UI 구성 요소, 그리고 기술적 구현 방안을 정의한다.

## 3. 핵심 기술 스택 및 사양 (Core Tech Stack)

### 3.1. Frontend (React)
- **Framework**: React.js (v18+)
- **HTTP Client**: **Axios**를 사용하여 Spring Boot API와 비동기 통신 수행.
- **State Management**: Context API 또는 Redux Toolkit을 통한 전역 상태(사용자 정보, 토큰) 관리.
- **Library**: `react-calendar-heatmap` (활동 시각화), `react-router-dom` (SPA 라우팅).

### 3.2. Backend (Spring Boot)
- **Framework**: **Spring Boot** (Java 17+)
- **Security**: Spring Security + JWT를 이용한 Stateless 인증 체계 구축.
- **Data Access**: Spring Data JPA를 사용하여 데이터베이스 조작.
- **API Design**: RESTful API 원칙 준수 (JSON 기반 응답).

### 3.3. Database (MySQL)
- **DBMS**: **MySQL 8.0**
- **Architecture**: 관계형 데이터 모델링을 통한 데이터 무결성 보장.
- **Key Tables**:
    - `User`: 학번(PK), 암호화된 비밀번호, 성명.
    - `Course`: 강의ID(PK), 강의명, 교수명.
    - `Note`: 노트ID(PK), 강의ID(FK), 학번(FK), 내용(TEXT), 수정일시.
    - `Post`: 게시물ID(PK), 강의ID(FK), 학번(FK), 제목, 내용, 작성일시, 좋아요수.
    - `Comment`: 댓글ID(PK), 게시물ID(FK), 학번(FK), 내용, 작성일시.

## 4. 비기능적 기술 상세
- **성능**: 히트맵 데이터 조회 시 `GROUP BY`와 `COUNT`를 사용하여 최적화된 쿼리 실행.
- **보안**: 비밀번호는 BCryptPasswordEncoder를 통해 암호화 저장.
- **사용성**: 모든 입력 폼에 적절한 Validation 및 Loading Spinner 적용.
