# [Uni Note] 요구사항 명세서 (Requirements Specification)

## 1. 프로젝트 개요
- **프로젝트명**: Uni Note (유니 노트)
- **목적**: 별도의 회원가입 절차 없이 학번과 비밀번호만으로 접근하여, 개인별 학습 활동(히트맵)을 관리하고 강의별 스마트 필기 및 익명 커뮤니티를 이용할 수 있는 통합 학습 대시보드 구축.

---

## 2. 사용자 요구사항 (User Requirements)

### 2.1. 학생 (Student)
- 번거로운 회원가입 없이 학교 시스템의 학번과 비밀번호(가정)로 즉시 로그인하고 싶다.
- 내가 이번 학기에 듣는 모든 강의를 대시보드 형태로 한눈에 확인하고 싶다.
- 나의 학습 성실도를 시각적으로 확인할 수 있는 활동 히트맵(GitHub 스타일)이 필요하다.
- 강의별로 최적화된 스마트 노트 템플릿에 빠르게 필기하고 싶다.
- 강의와 관련된 내용을 교수님이나 동기들 눈치 보지 않고 익명으로 자유롭게 질문하거나 소통하고 싶다.

## 4. 비기능 요구사항 (Non-Functional Requirements)

### 4.1. 보안 및 익명성 (Security & Anonymity)
- **익명성 유지**: 데이터베이스에는 작성자 정보가 기록되더라도, API 응답 시에는 이를 마스킹하여 클라이언트 단에서 작성자를 추적할 수 없게 한다.
- **데이터 보호**: 학번과 비밀번호는 안전하게 관리되어야 하며, 권한이 없는 강의 정보에는 접근할 수 없어야 한다.

### 4.2. 시각화 및 사용성 (Usability)
- **활동 시각화**: 히트맵 데이터 로딩 시 성능 저하가 없도록 쿼리를 최적화한다.
- **반응형 대시보드**: 태블릿과 PC 환경에서 대시보드 레이아웃이 유연하게 조절되어야 한다.

---

## 5. UI/UX 디자인 가이드라인
- **메인 페이지**: 상단에는 사용자 히트맵, 중앙에는 수평/수직 카드형 강의 리스트, 하단에는 전체 게시판 피드 배치.
- **노트 페이지**: 좌측에는 목차/날짜 리스트, 우측에는 넓은 필기 영역(Smart Editor).
- **게시판**: 에브리타임과 유사한 깔끔한 화이트 톤의 익명 게시판 UI.
