# [UniNote] 프로젝트 주간 보고서 (1~2주차)

## 1. 프로젝트 설계 (Design)
### 1.1. 요구사항 및 세부기능 명세
- **요구사항 명세**: 통합 대학 생활 지원 플랫폼의 핵심 기능(학습 관리, 커뮤니티, 편의 정보) 정의
- **세부기능 명세**: 스마트 필기 노트(자동 저장), 익명 게시판(좋아요/댓글), 강의 대시보드 등의 동작 시나리오 구체화

### 1.2. REST API 및 ERD 설계
- **REST API 설계**: `/api/auth`, `/api/dashboard`, `/api/courses` 등 기능별 엔드포인트 및 JSON 응답 규격 정의
- **ERD 설계**: `Student`, `Professor`, `Course`, `Enrollment` 간의 관계 모델링 및 데이터 정합성 규칙 수립

---

## 2. 데이터베이스 구축 (Database)
### 2.1. MySQL 기반 DB 구축
- **DBeaver 활용**: `uninote_db` 생성 및 설계된 ERD를 바탕으로 물리적 테이블 스키마 생성
- **테이블 구조**: PK/FK 제약 조건 및 `student_num`, `course_code` 등 고유 인덱스 설정

### 2.2. 테스트 데이터 관리
- **SQL 스크립트 구축**: 교수(5명), 실제 학생 데이터(정대영), 강의(6개), 수강신청 정보를 포함한 통합 삽입 스크립트 작성
- **데이터 정합성 검증**: DBeaver를 통한 테이블 간 연관 관계 및 조회 쿼리 테스트 완료

---

## 3. 프로젝트 환경 설정 (Setup)
### 3.1. 백엔드 (Spring Boot)
- **Spring Boot 3.x & Java 17**: Gradle 기반 프로젝트 빌드 및 의존성 관리 설정
- **Security & JWT**: Spring Security 프레임워크 도입 및 무상태(Stateless) 인증을 위한 JWT 환경 구축

### 3.2. 프론트엔드 (React)
- **Vite & React 19**: 최신 환경의 프론트엔드 스택 구성 및 `axios`, `react-router-dom` 설치
- **CORS 설정**: 프론트엔드(`:5173`)와 백엔드(`:8080`) 간의 원활한 통신을 위한 보안 정책 수립

---

## 4. 로그인 및 대시보드 구현 (Implementation)
### 4.1. 백엔드 (Backend)
- **Auth API**: `AuthController`, `AuthService`를 통한 JWT 발급 및 비밀번호 검증 로직 구현
- **Dashboard API**: 로그인 정보를 기반으로 한 맞춤형 수강 강의 목록 반환 로직 완성

### 4.2. 프론트엔드 (Frontend)
- **UI 리뉴얼**: 대학 포털 스타일의 블루/화이트 테마 및 사이드바 레이아웃 적용
- **인증 연동**: `axios` 인터셉터를 통한 토큰 자동 관리 및 로그인 상태에 따른 페이지 라우팅 구현
