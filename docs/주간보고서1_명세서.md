# [Uni Note] 프로젝트 기술 명세 및 코드 구조 상세 분석

본 문서는 'Uni Note' 프로젝트의 백엔드 시스템을 중심으로 한 코드 구조와 주요 로직, 그리고 프론트엔드 및 기타 자산의 구성 현황을 상세히 기록한다.

---

## 1. 백엔드(Backend) 폴더 및 코드 구조 상세

백엔드는 **Spring Boot 3.x** 기반의 계층형 아키텍처(Layered Architecture)를 채택하여 유지보수성과 확장성을 높였다.

### 📂 `com.uninote.backend` (Root Package)
- **`BackendApplication.java`**: 프로젝트의 시작점이다. `@SpringBootApplication` 어노테이션을 통해 스프링의 자동 설정, 컴포넌트 스캔 등을 활성화하며, 내장 톰캣 서버를 구동한다.

### 📂 `controller` (Web Layer)
외부의 HTTP 요청을 받고 응답을 반환하는 접점이다.
- **`AuthController.java`**: 로그인 요청을 처리한다.
  - `POST /api/auth/login`: `AuthService`를 호출하여 인증 결과(JWT 토큰)를 반환한다.
- **`DashboardController.java`**: 대시보드 화면에 필요한 정보를 제공한다.
  - `GET /api/dashboard/courses`: 현재 로그인한 학생의 수강 강의 목록을 반환한다.

### 📂 `service` (Business Layer)
실제 핵심 비즈니스 로직이 수행되는 곳이다.
- **`AuthService.java`**: 학번 존재 여부 확인 및 비밀번호 대조 로직을 포함한다.
  ```java
  // 대표 로직: 비밀번호 검증 (현재 평문 비교 방식)
  if (!loginRequest.getPassword().equals(student.getPassword())) {
      throw new RuntimeException("비밀번호가 일치하지 않습니다.");
  }
  ```
- **`DashboardService.java`**: DB에서 조회한 원본 데이터(Entity)를 프론트엔드에 최적화된 형태(DTO)로 변환하는 역할을 수행한다.

### 📂 `domain` (Persistence Layer - Entity)
데이터베이스의 테이블과 1:1로 매핑되는 객체들이다. JPA를 통해 관리된다.
- **`Student.java`**: 학생 정보(`studentNum`, `name`, `password` 등)를 정의한다.
- **`Course.java`**: 강의 정보(`courseCode`, `name`, `professor` 등)를 정의한다.
- **`Enrollment.java`**: 학생과 강의 간의 다대다(N:M) 관계를 매핑하는 수강신청 정보 테이블이다.

### 📂 `repository` (Data Access Layer)
데이터베이스에 접근하는 인터페이스이다. Spring Data JPA가 실제 구현체를 생성한다.
- **`StudentRepository.java`**: `findByStudentNum()`과 같은 사용자 정의 쿼리 메소드를 통해 DB 조회를 수행한다.

### 📂 `security` (Security & Authentication)
시스템의 보안과 사용자 인증(JWT)을 관리한다.
- **`SecurityConfig.java`**: Spring Security 설정 파일이다. CORS 허용 범위, CSRF 비활성화, API 접근 권한(PermitAll vs Authenticated)을 설정한다.
- **`JwtUtil.java`**: JWT 토큰의 생성(Generate), 파싱(Parse), 만료 여부 검증 등을 수행하는 유틸리티 클래스이다.
- **`JwtFilter.java`**: 모든 API 요청 헤더의 `Authorization` 토큰을 검사하여 유효한 사용자인지 판단하는 필터이다.

### 📂 `dto` (Data Transfer Object)
계층 간 데이터 이동을 위한 순수 객체이다. 보안상 엔티티를 직접 외부에 노출하지 않기 위해 사용한다.
- **`LoginRequest.java`**: 로그인 시 클라이언트가 보내는 JSON 데이터를 담는다.
- **`CourseResponse.java`**: 대시보드에 표시할 강의 정보를 정제하여 담는다.

## 2. 프론트엔드(Frontend) 및 기타 구성 요소

### 📂 `frontend/` (Vite + React)
사용자 인터페이스(UI)를 담당하며 백엔드 API와 통신한다.
- **`src/api/client.js`**: `axios` 인스턴스를 설정하고, `LocalStorage`에 저장된 JWT 토큰을 모든 요청 헤더에 자동으로 포함시킨다.
- **`src/pages/`**:
  - `LoginPage.jsx`: 학번/비밀번호 입력 및 토큰 저장 로직.
  - `DashboardPage.jsx`: 백엔드로부터 받은 강의 목록을 카드 형태로 렌더링.
- **`package.json`**: `react-router-dom`, `axios` 등의 라이브러리 의존성이 정의되어 있다.

### 📂 `docs/` (Documentation)
프로젝트의 설계 및 진행 과정을 기록한 문서 저장소이다.
- `요구사항 명세서.md`: 시스템이 갖춰야 할 핵심 기능 정의.
- `rest api 설계.md`: 프론트와 백엔드 간의 데이터 통신 규약.
- `db구조.md`: 테이블 간의 관계(ERD) 및 컬럼 정의.
- `0401_x.md`: 일자별 개발 이슈 및 해결 과정 기록.

### 📂 `ui이미지/`
- 기획 단계에서 생성한 UI/UX 디자인 시안들이 보관되어 있으며, 개발 시 컴포넌트 스타일링의 기준점이 된다.

---

## 3. 시스템 흐름 요약 (Workflow)

1. **로그인**: `LoginPage` -> `AuthController` -> `AuthService` (DB 조회) -> JWT 발급 -> `client.js` 토큰 저장.
2. **데이터 조회**: `DashboardPage` -> `DashboardController` (JWT 검증) -> `DashboardService` -> `CourseRepository` -> `CourseResponse` 반환 -> 화면 렌더링.
3. **보안**: 모든 요청은 `JwtFilter`를 거쳐 토큰의 유효성을 검증받아야만 자원에 접근 가능하다.
